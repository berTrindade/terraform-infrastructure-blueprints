# Contributing Guidelines

Thank you for considering contributing to this project!

## How to Contribute

1. Clone the repository and create a new branch for your changes.
2. Make your changes, following our code style and documentation standards.
3. Ensure all CI checks pass before submitting your changes.
4. Submit a pull request with a clear description of your changes and why they are useful.
5. Respond to review feedback and update your PR as needed.

## Code Style & Standards

- Use conventional commit messages (e.g., `feat: ...`, `fix: ...`, `docs: ...`).
- Write clear, self-contained Terraform modules and documentation.
- Avoid dependencies on internal or proprietary systems.

## Review Process

- All changes are reviewed by the maintainer or designated code owner.
- You may be asked to make changes before your PR is merged.
